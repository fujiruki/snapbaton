# 05. 技術設計

## プラグイン構成

```
snapbaton/
  snapbaton.php           ← メインプラグインファイル
  includes/
    class-activator.php   ← テーブル作成・初期化
    class-admin.php       ← 管理画面の登録
    class-rest-api.php    ← REST APIエンドポイント
    class-permissions.php ← 権限管理
  admin/
    build/                ← React ビルド出力
    src/                  ← React ソース（管理画面UI）
    package.json
    vite.config.ts
  assets/
    css/
    js/
  languages/              ← 翻訳ファイル（日本語・英語）
  readme.txt              ← WordPress.org用
```

## 技術スタック

| レイヤー | 技術 |
|:--|:--|
| プラグイン本体 | PHP 8.0+ |
| 管理画面UI | React + TypeScript（WP管理画面内に埋め込み） |
| ビルドツール | Vite（または @wordpress/scripts） |
| 画像編集 | `@fujiruki/react-image-editor-lightbox`（検討中） |
| REST API | WordPress REST API を拡張 |
| DB | WordPress標準DB + カスタムテーブル |

## 画像保存方針

- WPメディアライブラリを利用（`wp_insert_attachment`）
- オリジナルサイズを保持するため、アップロード時に `big_image_size_threshold` を無効化
- サムネイルはWP標準の画像サイズ生成を利用
- SnapBaton独自のメタデータ（タイトル・説明文）はカスタムテーブルで管理

## EXIF操作

- PHP `exif_read_data()` でアップロード時に読み取り
- PEL（PHP Exif Library）等で書き込み（要ライブラリ選定）

## GitHub公開方針

- リポジトリ: `github.com/fujiruki-dev/snapbaton`（仮）
- ライセンス: GPL v2（WordPress準拠）
- WordPress.orgへの公開も視野に入れる
- README.md / CONTRIBUTING.md を整備
- GitHub Actions でリリースZIP自動生成

## 国際化

- `__()` / `_e()` でテキスト国際化
- 日本語と英語に対応
- テキストドメイン: `snapbaton`
